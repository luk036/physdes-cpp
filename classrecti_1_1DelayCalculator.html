recti::DelayCalculator class

Abstract base class for delay calculation models.

Contents

  • Reference
    • Derived classes
    • Constructors, destructors, conversion operators
    • Public functions

This class defines the interface for different delay models (e.g., Linear, Elmore) used in the DME algorithm to estimate wire delays and capacitances. Concrete implementations must provide methods for calculating wire delay, delay per unit, wire capacitance, and the optimal tapping point for merging subtrees.

std::pair<int, double> recti::DelayCalculator::calculate_tapping_point(TreeNode& node_left, TreeNode& node_right, int distance) pure virtual

Determines the optimal tapping point for merging two subtrees to achieve zero skew.

Parameters
node_left The left child node (subtree root).
node_right The right child node (subtree root).
distance The Manhattan distance between the merging segments of node_left and node_right.
Returns

A pair containing:

  • The extend_left value: the length of the wire segment from node_left's merging segment to the tapping point.
  • The delay_left value: the delay from the clock source to the tapping point.

This method calculates where a new parent node should be placed on the merging segment between two child nodes to balance their delays. It also returns the delay at this new tapping point.
